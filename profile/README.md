## What we work on

AI agents are capable but amnesiac: every task starts from scratch, and most "memory"
is a wiki that goes stale the moment a fact changes. We build the missing layer — **the
brain**, a bitemporal knowledge graph that updates as facts change and keeps a clean
version history instead of accumulating contradictions. Humans and agents read from and
write to the same memory.

Around it we build an **AI-native workspace** — chat, docs, mail, calendar, tasks, and
CRM — where every surface shares that memory and agents act with the same context the
team sees.

## Open research & tools

We publish our methods and tools, including the parts where we don't win:

- **[unison-evals](https://github.com/Unison-Workspace/Unison-evals)** — an open
  benchmark suite for **AI memory and agent systems**. Reproducible methodology;
  results reported in full, wins and losses alike. Ships **BitempoQA**, an open dataset
  for bitemporal (time-aware) reasoning over changing facts.
- **[Brain client](https://github.com/Unison-Workspace/unison-brain)** — an
  open-source **CLI, SDK, and MCP server** so any coding agent can read and write a
  shared memory.

*Both are rolling out publicly — watch this org.*

## How we build

Memory is only useful if it's trustworthy. Agents see only what you connect; Unison
never trains on your data and never shares it across workspaces. Encrypted in transit
and at rest, hosted on EU infrastructure, GDPR-aligned by design.
